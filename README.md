### Командная строка

Является текстовым интерфейсом для работы с программами и файловой системой. Находясь в ней,  
мы всегда находимся в какой-то папке. С помощью нее мы работаем с Git'ом.

_Полезные команды_  <br>
_~_ - таким образом обозначается домашняя директория (это папка Users/userName). То есть папка текущего пользователя <br>
_/c_ - корень файловой системы <br>
_pwd_ (print working directory) - вывести название текущей директории (в которой находится пользователь) <br>
_ls_ - команда для отображения содержимого папки. Можно применить ключ -a для того, чтобы  <br> 
посмотреть в том числе скрытые файлы, а ключ -l выводит списком содержимое с кратким описанием <br>
_cd_ путь (change directory) - поменять текущую директорию <br>
_.._ - обозначение родительской папки <br>
_._ - текущая папка <br>
_touch_ имя_файла.расширение - создать файл <br>
_cat_ имя_файла - вывести содержимое файла (только текстовые) <br>
_mkdir_ - создать директорию, с ключом -p можно создать последовательно вложенные директории <br>
_rm_ - удалить файл <br>
_rmdir_ - удалить _пустую_ директорию <br>
_rm -r_ - ключ -r говорит о рекурсивности удаления, т.е. все файлы и папки в той папке, которая указывается для удаления <br>


### Git

Система контроля версий. Git - программа, дающая возможность следить за изменениями файлов и записывать их  
Он позволяет работать с локальной версией репозитория и предоставляет возможность работать с удаленной версией.

_Команды, которые стоит запомнить_ <br>
_git config_ - это настройки Git'a. Папка .gitconfig находится обычно в домашней директории.
_git log_ - показывает лог коммитов
_git config --global_ - позволяет задать настройки глобально, например, user.name "user name".
_git init_ - Создает Git-репозиторий. Она создает в текущей папке папку .git, которая хранит всю историю изменений. При ее удалении  
вся история потеряется.
_rm -rf .git_ - команда, которая позволяет "разгитить" текущую папку, если мы вдруг случайно сделали папку Git-репозиторием.
_git add имя_файла (git add .  | git add --all)_ - позволяет _зафиксировать_ (но не записать) изменения файлов.
_git commit -m "message"_ - коммит, то есть сохранение/запись изменений.
_git remote_ - проверить с кем есть удаленное подключение у этого Git-репозитория. С ключом -v показывает чуть больше информации.
_git remote add nameProj linkRepositoryGitHub_ - команда, благодаря которой мы устанавливаем _первый раз соединение_ с удаленным репозиторием  
ключ -u позволяет синхронизировать ветки.
_git push -u псевдоним_проекта главная_ветка_ - Первый коммит в удаленную версию всегда задается такой командой.  псевдоним проекта мы задаем предыдущей командой, когда создаем удаленное подключение. Главная ветка  
это имена main или master. git push - команда для всех последующих коммитов
_git push_ - команда, которая записывает изменения из локального репозитория на удаленный

----

Хеш коммита - это уникальный идентификатор коммита. По нему можно найти необходимый коммит  
В папке .git есть список хешей а так же таблица соответствия "хеш -> информация о коммите". Под информацией о коммите подразумевается  
кто внес изменения, когда, содержимое файлов и ссылка на прошлый (родительский коммит)  
HEAD -> master. Это файл в папке .git, который содержит ссылку на служебный файл с хешем последнего коммита, когда мы делаем коммит, этот файл обновляется.  

Статусы файлов: неотслеживаемый, отслеживаемый, модифицированный и подготовленный к коммиту  
неотслеживаемый: такой статус присваивается файлу, который был создан/добавлен в репозитории, но еще не был добавлен в staging area (с помощью команды git add name_file)  
отслеживаемый: файлы, которые попадали (или сейчас там находятся) в staging area  
модифицированный: файл, версия которого не зафиксирована, т.е. версия файла не находится в списке на коммит (staging area)  
подготовленный к коммиту (есть в списке на коммит): файл, добавленный через git add

   create new file --> untracked --> "git add file_name" --> staged, tracked  
                       staged --> "git commit -m "message" --> tracked/commited;
                       modified --> изменение отслеживаемого файла
                       tracked --> файл, который попадал в staged

### GitHub 

Платформа, позволяющая хранить репозиторий удаленно для разных целей, например: для совместной работы над проектом; для того, чтобы с кем-то поделиться проектом.

_Настройка и подключение_  
Создаем репозиторий на вкладке "Repository", нажав зеленую кнопку "New".  
Следуем инструкции: заполняем имя и так далее. Далее нужно указать SSH-ключ в настройках.  
С помощью этого ключа данные будут шифроваться
