### Командная строка

Является текстовым интерфейсом для работы с программами и файловой системой. Находясь в ней  
мы всегда находимся в какой-то папке. С помощью нее мы работаем с Git'ом.

_Полезные команды_  <br>
~ - таким образом обозначается домашняя директория (это папка Users/userName). То есть папка текущего пользователя
/c - корень файловой системы
pwd (print working directory) - вывести название текущей директории (в которой находится пользователь)
ls - команда для отображения содержимого папки. Можно применить ключ -a для того, чтобы  
посмотреть в том числе скрытые файлы, а ключ -l выводит списком содержимое с кратким описанием
cd путь (change directory) - поменять текущую директорию
.. - обозначение родительской папки
. - текущая папка
touch имя_файла.расширение - создать файл
cat имя_файла - вывести содержимое файла (только текстовые)
mkdir - создать директорию, с ключом -p можно создать последовательно вложенные директории
rm - удалить файл
rmdir - удалить _пустую_ директорию
rm -r - ключ -r говорит о рекурсивности удаления, т.е. все файлы и папки в той папке, которая указывается для удаления


### Git

Система контроля версий. Git - программа, дающая возможность следить, записывать все изменения продукта.  
Он позволяет работать с локальной версией репозитория и предоставляет возможность работать с удаленной версией.

_Команды, которые стоит запомнить_ <br>
git config - это настройки Git'a. Папка .gitconfig находится обычно в домашней директории.
git config --global - позволяет задать настройки глобально, например, user.name "user name".
git init - Создает Git-репозиторий. Она создает в текущей папке папку .git, которая хранит всю историю изменений. При ее удалении  
вся история потеряется.
rm -rf .git - команда, которая позволяет "разгитить" текущую папку, если мы вдруг случайно сделали папку Git-репозиторием.
git add имя_файла (git add .  | git add --all) - позволяет _зафиксировать_ (но не записать) изменения файлов.
git commit -m "message" - коммит, то есть сохранение/запись изменений.
git remote - проверить с кем есть удаленное подключение у этого Git-репозитория. С ключом -v показывает чуть больше информации.
git remote add nameProj linkRepositoryGitHub - команда, благодаря которой мы устанавливаем _первый раз соединение_ с удаленным репозиторием  
ключ -u позволяет синхронизировать ветки.
git push -u псевдоним_проекта главная_ветка - Первый коммит в удаленную версию всегда задается такой командой.  псевдоним проекта мы задаем предыдущей командой, когда создаем удаленное подключение. Главная ветка  
это имена main или master. git push - команда для всех последующих коммитов
git push - 
### GitHub 

Платформа, позволяющая хранить репозиторий удаленно для разных целей, например: для совместной работы над проектом; для того, чтобы с кем-то поделиться проектом.

_Настройка и подключение_  
Создаем репозиторий на вкладке "Repository", нажав зеленую кнопку "New".  
Следуем инструкции: заполняем имя и так далее. Далее нужно указать SSH-ключ в настройках.  
С помощью этого ключа данные будут шифроваться
